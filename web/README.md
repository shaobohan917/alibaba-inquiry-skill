# Web 可视化控制系统

## 🚀 快速启动

### 方法一：使用 npm（推荐）

```bash
cd web
npm start
```

### 方法二：直接运行 Node.js

```bash
cd web
node server.js
```

### 方法三：使用 npx

```bash
cd web
npx http-server -p 8080
```

---

## 📋 功能说明

### 1. 总监管指挥中心

启动服务器后，浏览器打开 `http://localhost:3000`，您可以：

#### 👔 启动总监管 Agent
- 点击「🚀 启动总监管」按钮
- 总监管会自动创建今日任务并分发到任务队列
- 输出日志显示任务创建详情

#### ⚙️ 启动任务执行器
- 点击「▶️ 启动」按钮
- 任务执行器开始轮询任务队列（每 5 秒）
- 自动发现 pending 任务并启动对应 Agent 执行
- 实时显示执行日志

#### 📊 任务状态概览
- 总计 / 待处理 / 进行中 / 已完成 / 失败
- 每 5 秒自动刷新
- 手动点击「🔄 刷新状态」立即更新

## 🔌 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/supervisor/start` | POST | 启动总监管 Agent |
| `/api/executor/start` | POST | 启动任务执行器 |
| `/api/executor/stop` | POST | 停止任务执行器 |
| `/api/tasks/status` | GET | 获取任务状态统计 |
| `/api/executor/status` | GET | 获取执行器运行状态 |

## 🎮 使用流程

### 完整自动化流程

1. **启动 Web 服务器**
   ```bash
   cd web
   npm start
   ```

2. **打开浏览器**
   ```
   http://localhost:3000
   ```

3. **启动总监管**
   - 点击「🚀 启动总监管」
   - 等待 3 秒，查看输出日志
   - 任务已创建到 `data/tasks/` 目录

4. **启动任务执行器**
   - 点击「▶️ 启动」
   - 等待 5 秒，查看执行日志
   - 任务执行器自动发现并执行任务

5. **查看任务进度**
   - 任务状态概览自动刷新
   - 查看各状态任务数量
   - 日志区域显示详细输出

## ⚠️ 注意事项

1. **确保 Chrome 已安装**：自动化操作需要 Chrome 浏览器
2. **首次运行需要登录**：Cookie 会自动保存，后续自动登录
3. **保持服务器运行**：关闭服务器会终止任务执行器
4. **端口占用**：默认使用 3000 端口，如有冲突请修改

## 🐛 故障排查

### 服务器启动失败
```bash
# 检查端口占用
lsof -i :3000

# 杀死占用端口的进程
kill -9 <PID>
```

### API 调用失败
- 检查服务器是否运行
- 浏览器控制台查看 CORS 错误
- 确认 `localhost:3000` 可访问

### 任务执行失败
- 检查 `data/tasks/` 目录权限
- 查看 Agent 日志输出
- 确认 Chrome 浏览器可访问
