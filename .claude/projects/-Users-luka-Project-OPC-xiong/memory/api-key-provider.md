---

**商业模式**：API Key 由提供方（用户）统一配置，客户付费购买软件使用权。类似传统软件的授权模式。

**Why**：用户选择方案 B，客户是技术小白且不应承担 API 成本，这样产品可以做成一价全包，用户体验更简单。

**How to apply**：产品设计需包含：1) 软件授权/激活机制（防止滥用）2) 使用次数/期限管理 3) 后台 API Key 统一管理 4) 可能的话做离线授权验证。
